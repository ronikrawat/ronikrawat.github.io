General GitHub Questions

  • Q: What is GitHub, and why is it used?
  • A: GitHub is a web-based platform for version control and collaboration, allowing multiple developers to work on projects together. It hosts Git repositories and provides additional features like pull requests, issues, and project management tools.
  • Q: Explain the difference between Git and GitHub.
  • A: Git is a version control system used to track changes in source code during software development. GitHub is a cloud-based platform that hosts Git repositories and provides tools for collaboration, such as issue tracking, code reviews, and more.

GitHub Commands Questions

  • Q: How do you initialize a new Git repository?
  • A: By using the git init command, which creates a new Git repository in the current directory.
  • Q: How can you undo a commit in Git?
  • A: You can use git revert commit_id to create a new commit that undoes the changes from a previous commit, or git reset --soft commit_id to move the HEAD pointer to a previous commit without modifying the working directory.
  • Q: What is the difference between git merge and git rebase?
  • A: git merge combines the changes from two branches into one, creating a new merge commit. git rebase moves or combines a sequence of commits to a new base commit, resulting in a linear history.
  • Q: How do you remove a file from Git without deleting it from your local file system?
  • A: You can use git rm --cached filename to remove the file from the Git index, so it won't be tracked, but it remains in your local file system.

Collaboration and Workflow Questions

  • Q: What is a pull request, and how does it work?
  • A: A pull request is a feature in GitHub that allows developers to notify team members that they have completed a feature or bug fix and request it to be reviewed and merged into the main codebase. It helps in code reviews and discussion before the changes are merged.
  • Q: How can you resolve a merge conflict in Git?
  • A: A merge conflict occurs when Git cannot automatically merge changes. To resolve it, open the conflicting file, manually edit the content to resolve the conflict, stage the changes using git add, and then commit the merge.
  • Q: What is the significance of forking a repository on GitHub?
  • A: Forking a repository creates a personal copy of someone else's repository. It allows you to freely experiment with changes without affecting the original project. Once changes are made, you can submit a pull request to propose merging your changes into the original repository.

Advanced GitHub Questions

  • Q: Explain the concept of Git stash and when you would use it.
  • A: Git stash temporarily saves changes in your working directory that are not yet ready to be committed, allowing you to switch branches or perform other operations without losing your changes. You can later apply the stashed changes to any branch using git stash apply.
  • Q: How do you force push to a remote repository, and when is it appropriate?
  • A: Use git push --force to forcefully push changes to a remote repository, which can overwrite commits. This is appropriate when you need to rewrite the commit history, but it should be used cautiously to avoid overwriting others' work.
  • Q: What is GitHub Actions, and how does it work?
  • A: GitHub Actions is a CI/CD platform that allows you to automate tasks like testing, building, and deploying your code directly from GitHub repositories. You define workflows in YAML files, which are triggered by GitHub events such as pushes or pull requests.
